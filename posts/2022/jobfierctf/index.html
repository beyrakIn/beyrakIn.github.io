I try mrrafael as a username and password as a password. Then I did catch the request with BurpSuite. As you can see there are username (2) and password (1) fields. I sent this request to intruder.

I know username, I need just to find password. That’s why I add only password field to brute-force.

Every wrong passwords return 302(1) http status code (redirection), probably correct password also returns 302. This response also returns path (2) with Location header. Wrong passwords Location value is /accounts/login/. The correct password will return different Location value. that’s why we should follow redirection to analyze password is correct or not.
